## 🚦 Validation Rules (The "Brain")
The application must check these rules before adding a card. If a rule is violated, use `alert()` to notify the user and **stop** the addition.

| Feature | Validation Rule |
| :--- | :--- |
| **Parental Limit** | There can only be **one** Father and **one** Mother in the system. |
| **Sibling Limit** | Maximum of **7 kids** (Sons/Daughters combined) allowed. |
| **Hierarchy** | Children **must** be younger than the youngest parent currently in the list. |
| **Legal Constraint** | If "Married" is selected, the member's age must be **18 or older**. |
| **Sanity Check** | Age must be a positive number greater than 0. |

## 🪪 The Member Card

Each member added to the directory must be represented by a dynamic "Profile Card." This card serves as the visual output of your data objects.

Card Requirements:

- **Role-Based Styling:** Use CSS classes to color-code cards (e.g., Green borders for parents, Blue for children).
- **Data Display:** Clearly show the member's Name, Role, Age, and Status.
- **Conditional Indicators:** Display a specific icon or text tag (e.g., "💼 Working", '🏠 Not Working') only if the isWorking checkbox was checked.
- **Functional Removal:** Every card must contain a Delete button that removes that specific card from the UI and clears the associated data from your JavaScript array.

## 💻 Technical Tasks

### 1: DOM Selection & Listeners
Select all form inputs and the grid container using `document.querySelector`. Attach a `submit` event listener to the form and use `event.preventDefault()` to manage the submission manually. Do not reset the form after each member addition.

### 2: State Tracking
Maintain an array of objects to keep track of the current family members.
```javascript
let familyMembers = [];
```

This array is essential for calculating the number of kids or checking the ages of parents already in the list.

### 3: Dynamic Card Rendering
When a user is added, create a new HTML element and append it to the grid following the correct coloring.

### 4: Removal Logic
Every card must have a "Remove" button. When clicked:

- The card must be removed from the DOM.
- The corresponding member must be removed from the Data Array.
